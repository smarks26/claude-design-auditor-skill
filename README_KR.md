# 🎨 Design Auditor — Claude 스킬

**19가지 전문 디자인 카테고리**를 기준으로 디자인을 감사하는 Claude 스킬입니다. 개발자, 비디자이너, 디자인 규칙을 몰라도 전문적인 피드백이 필요한 모든 분을 위해 만들어졌습니다.

**Figma 파일** (Figma MCP), **코드** (HTML/CSS/React/Vue), **스크린샷**, **와이어프레임**, **라이브 URL**, **GitHub 링크** 등 다양한 입력을 지원합니다. **한국어와 영어** 모두 지원합니다.

**Claude**, **Manus**, 그 외 SKILL.md 기반 스킬을 지원하는 에이전트와 호환됩니다.

---

## 무엇을 해주나요?

Figma 링크를 붙여넣거나, CSS를 공유하거나, 스크린샷을 업로드하거나, 와이어프레임을 첨부하면 — Design Auditor가 19가지 디자인 규칙 카테고리에 따라 검토하고 다음을 제공합니다:

- **100점 만점 점수** + 카테고리별 세부 점수 (각 행에 미니 바 차트 포함)
- 별도 **접근성 점수** (WCAG 기준 — Cat 2, 6, 7, 15, 16)
- 별도 **윤리 점수** (다크 패턴 — Cat 18)
- 별도 **사용성 점수** (닐슨 휴리스틱 — Cat 19, H4/H5/H8/H9 교차 참조 포함)
- **🚫 차단** 레벨 — WCAG AA, GDPR, PECR 등 법적 위반 사항 (−12점)
- 심각도별 이슈 분류 (🚫 차단 / 🔴 심각 / 🟡 경고 / 🟢 팁)
- **색맹 컨텍스트** — 실패한 색상 조합마다 영향 받는 색맹 유형 주석 추가
- **SVG 접근성 검사** — 장식/의미 SVG 패턴, aria-hidden, role="img"
- **디자인 시스템 감지** — MUI, Chakra, shadcn/ui, Ant Design, Radix, Bootstrap (시스템별 수정 방법 제시)
- **Figma 오토 레이아웃 스캔** — 오토 레이아웃 대신 수동 배치 사용 중인 프레임 감지
- 각 규칙이 **왜** 중요한지 쉬운 언어로 설명
- **"원리 설명(Teach Me)" 모드** — 수정 방법이 아닌 디자인 원칙 중심의 3레슨 설명
- 이슈를 수정 난이도 vs 개선 효과로 매핑한 **이슈 우선순위 매트릭스**
- HTML/CSS/React/Vue 코드의 **수정 전/후 코드 diff**
- Figma MCP를 통한 **Figma 파일 직접 수정**
- Figma Code Connect — 디자인-코드 간 매핑 갭 감지
- **와이어프레임 → 스펙** 변환 — 개발 즉시 사용 가능한 주석 스펙 생성
- **개발자 전달 보고서** — CSS 스펙 표, 접근성 체크리스트, 핵심 수정사항
- **Canva 시각적 보고서** — 이해관계자와 공유 가능한 시각 요약
- **마크다운 보고서 내보내기** — Notion, GitHub, Linear, Jira 활용 가능

## 누구를 위한 스킬인가요?

- 디자인 배경이 없는 **개발자**
- 빠른 피드백 또는 WCAG 확인이 필요한 **디자이너**
- Figma MCP 또는 VS Code에서 디자인 검증을 원하는 **팀**
- 윤리적이고 접근 가능한 제품을 만들고 싶은 **PM 및 창업자**
- "이거 괜찮아 보여?"가 궁금한 **모든 분**

## 사용 방법

설치 후 Claude에게 자연스럽게 말하면 됩니다:

```
"디자인 검토해줘" → 범위 선택 후 전체 감사
"접근성 확인해줘" → 접근성 중심 감사
"이 폼 좀 봐줘" → 관련 카테고리 부분 감사
"WCAG 기준에 맞아?" → 대비 및 접근성 감사
"이 Figma 파일 확인해줘: [링크]" → Figma MCP 감사
"다크 패턴 있어?" → 윤리 감사
"와이어프레임 스펙 출력해줘" → 와이어프레임 → 스펙 모드
"휴리스틱 검토해줘" → 닐슨 사용성 감사
"이거 GDPR 준수해?" → 윤리 및 접근성 감사
"색상 대비 확인해줘" → 색상 및 접근성 감사
```

Figma URL, GitHub 링크, 라이브 사이트 URL, 스크린샷, HTML/CSS 코드를 공유하면 Claude가 자동으로 감사를 실행하고 입력한 언어로 응답합니다.

## 지원하는 URL 유형

| URL 유형 | 신뢰도 | 처리 방식 |
|---|---|---|
| 라이브 웹사이트 (`https://...`) | 🟡 보통 | web_fetch, 로그인 처리, 다중 페이지 제안 |
| Vercel / Netlify 프리뷰 | 🟡 보통 | 라이브 웹사이트와 동일 |
| GitHub 파일 URL | 🟢 높음 | 원시 소스 가져와 전체 코드 감사 |
| GitHub 저장소 URL | 🟡 보통 | 주요 파일 탐색, 범위 선택 위젯 |
| CodeSandbox / StackBlitz | 🟡–🟢 | 소스 접근 가능 시 코드 감사, 아니면 미리보기 |
| CodePen | 🟡 보통 | 디버그 URL 가져오기, 소스 추출 시도 |
| Storybook | 🟡 보통 | URL 경로에서 컴포넌트/변형 감지 |

## 변경 이력

### v1.2.12

**nudges.fyi 행동 패턴 추가, 버그 수정, 시각적 보고서 개선.**

**신규: 5가지 행동 패턴 추가**
- **선택 과부하(Overchoice Paradox) → Cat 4** — 주요 액션 구분 없이 4개 이상 동일 비중의 선택지 → 🟡; 6개 이상 → 🔴. 목록·내비·테이블 제외
- **피크-엔드 법칙(Peak-End Rule) → Cat 11** — 성공/완료 화면의 기억 가능성 감사: 완료 내용, 다음 단계, 시각적 강화 요소, 자동 리다이렉트 타이밍
- **목표 경사 이론(Goal Gradient Theory) → Cat 11** — 진행 바 없는 다단계 플로우, 정적 진행 바, 전체 단계 수 미표시, `aria-valuenow` 누락 감지
- **디코이 가격 책정(Decoy Pricing) → Cat 18 Group A** — 3열 가격 표에서 모든 기준에서 열등한 하나의 등급 감지
- **조작적 앵커링(Manipulative Anchoring) → Cat 18 Group D** — `text-decoration: line-through` / `.was-price` / `.rrp` 패턴 감지

**버그 수정**
- Cat 18 패턴 수 업데이트: 20개 → 22개 (두 README 모두)
- Cat 6 SVG `<use>`: `href`와 함께 `xlink:href` 레거시 검사 추가
- Cat 19 H1 "멀티스텝 진행 표시": Cat 11 목표 경사 이론 교차 참조 추가로 중복 플래그 방지
- 점수 패널: 윤리 점수 별도 산식 주의 문구 추가 — `🔴 −15 · 🟡 −7 · 🟢 0`
- `heuristics.md` 구분자: `-` → `—`
- README Cat 11 행: 피크-엔드 법칙 및 목표 경사 이론 명시

**시각적 보고서 개선**
- 레이더 차트: 이진 보라/빨강 → 3색 스케일 — 보라(8–10) · 황색(6–7) · 빨강(≤ 5)
- 레이더 차트: Visualizer 사용 불가 시 카테고리 점수 표로 폴백 (영문 + 한국어)
- 이슈 우선순위 매트릭스: 한국어 축 라벨 추가 — `수정 난이도` (x) · `개선 효과` (y)
- 카테고리별 점수 표: 점수 밴드 각주 추가 — `≥ 8 ✅ · 5–7 🟡 · ≤ 4 🔴` (영문 + 한국어)
- 재감사 델타: Visualizer로 이전/이후 바 쌍 렌더링, 색상 코드 델타 배지 포함
- "원리 설명" 모드: 레슨 후 3카드 Visualizer 레이아웃, 각 카드 `sendPrompt()` 클릭 가능
